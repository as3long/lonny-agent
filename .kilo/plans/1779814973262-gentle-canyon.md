# TUI Display Optimization Plan

## Current State Analysis

The current TUI in `src/tui/index.ts` features:
- Split layout (chat left, plans/todos right) with custom `SplitLayout` component
- Markdown chat display with theme
- SelectList for plans with custom wrapper
- Text display for todos
- Input component at bottom
- Status bar showing mode, running state, and plan count
- Arrow key navigation for plans (up/down)
- Slash commands (/exit, /mode)

## Identified Issues

### 1. Layout Problems
- Side panel disappears abruptly when terminal < 100 columns (hard threshold)
- No padding/margins between sections
- Status bar uses plain Text without visual distinction

### 2. Visual Styling
- Headers lack background styling or visual prominence
- No separator between chat and side panel
- Todo list and plan list lack visual grouping
- Markdown code blocks have minimal styling

### 3. User Experience
- No search/filter capability in plans list (SelectList supports `setFilter`)
- Plan selection doesn't show full description effectively
- No visual feedback for running state beyond status text
- Todo items could use better formatting

### 4. Performance
- `setInterval` for resize polling is inefficient (500ms polling)
- Plans are reloaded on every selection change (redundant)
- No lazy loading for large plan files

## Proposed Optimizations

### Phase 1: Layout Improvements
- [ ] Use `Container` spacing with separators instead of custom SplitLayout
- [ ] Add visual separator (vertical bar or spacing) between panels
- [ ] Implement smooth responsive threshold (gradual width adjustment vs. binary hide/show)
- [ ] Add padding around all components for better readability

### Phase 2: Visual Styling
- [ ] Add background colors to headers using Box with bgFn
- [ ] Style status bar with distinct background (e.g., dimmed)
- [ ] Improve markdown theme with better contrast for code blocks
- [ ] Add visual indicators for todo completion (already has checkmarks, enhance colors)

### Phase 3: Enhanced Features
- [ ] Add search/filter functionality to plans list with `/` key binding
- [ ] Show plan file modification time in description
- [ ] Add lazy loading for plan content
- [ ] Implement proper resize event handling (reduce polling overhead)

### Phase 4: UX Improvements
- [ ] Add placeholder text to input ("Type message or /command...")
- [ ] Show spinner or progress indicator when running
- [ ] Add keyboard help overlay (press `?` to show)
- [ ] Better error message formatting

## Implementation Order

1. Refactor layout to use Container-based approach with separators
2. Add visual styling with background colors for headers
3. Implement search/filter for plans (press `/` to filter)
4. Optimize resize handling
5. Add visual feedback for running state
6. Add help overlay for keyboard shortcuts

## Files to Modify

- `src/tui/index.ts` - Main TUI implementation
